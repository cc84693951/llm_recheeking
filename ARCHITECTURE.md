# 模拟小模型检测送检大模型复判系统 - 架构与使用文档

## 一、系统概述

本系统是一款基于 PyQt5 的桌面应用，用于模拟"小模型检测框选目标 → 裁剪后送检大模型 → 复判结果回写标注"的完整工作流。系统强调**稳定性、通用性、可扩展性**，采用模块化分层架构，支持云端（硅基流动）与本地（OpenAI 兼容接口）两种大模型调用方式。

## 三、核心模块详解

### 3.1 image_manager.py
- **职责**：封装 Pillow 图像操作，隔离 UI 与图像处理细节。
- **功能**：
  - `load(path)`：加载图片并转为 RGB。
  - `rotate(angle)` / `resize(w, h)`：图像变换。
  - `crop(x, y, w, h)`：按原图像素坐标裁剪 ROI。
  - `to_base64(pil_image)`：将裁剪图转为 Base64 DataURL，供 LLM API 使用。
  - `draw_bboxes(...)`：在原图上绘制检测框、扩展虚线框、结果标签，返回 PIL.Image。

### 3.2 bbox_manager.py
- **职责**：管理所有检测框（BBox）的生命周期。
- **数据结构**：`BBox(x, y, w, h, label, result, expand_meta)`
- **核心算法**：`expanded_coords(img_w, img_h, ...)`
  - 支持**中心**、**上**、**下**、**左**、**右**五个方向的独立扩展。
  - 扩展逻辑：以原框为基准，向选中方向外扩。例如"上"扩展 2 倍，则新高度向上增加 `原高 × (2−1)`。
  - 最终通过 `clamp` 确保坐标不超出图像边界。

### 3.3 llm_client.py
- **职责**：统一封装大模型调用接口。
- **接口**：`inspect(pil_image, system_prompt, user_prompt, few_shots, params)`
- **协议**：兼容 OpenAI Chat Completions 格式。
  - 云端：硅基流动 API（`https://api.siliconflow.cn/v1`）
  - 本地：任意 OpenAI 兼容服务（如 ollama、vLLM）
- **参数**：temperature、top_p、max_tokens、frequency_penalty、presence_penalty。

### 3.4 model_fetcher.py
- **职责**：动态获取硅基流动模型广场列表。
- **容错**：若网络失败或接口变更，自动回退到内置常用模型列表（如 Qwen2-VL、InternVL2 等）。用户始终可手动输入模型 ID。

### 3.5 inspection_engine.py
- **职责**：异步送检引擎，继承 `QThread`，防止阻塞 UI。
- **流程**：
  1. 遍历当前所有 BBox。
  2. 根据扩展倍数计算 ROI 坐标并裁剪。
  3. 调用 `llm_client.inspect()` 送检。
  4. 发射 `progress`、`result`、`finished_signal` / `error` 信号。
- **中断**：支持用户点击"取消"来停止送检线程。

## 四、UI 组件详解

### 4.1 image_canvas.py（图像画布）
- **基类**：`QGraphicsView` + `QGraphicsScene`
- **核心功能**：
  - **图片显示**：`QGraphicsPixmapItem` 承载原图，scene 尺寸与图像像素 1:1 对应。
  - **缩放**：鼠标滚轮前后缩放 view 的 transform（1.15 倍步长）。
  - **平移**：中键拖拽或滚动条模式。
  - **绘制矩形框**：在空白处左键拖拽，生成新 BBox；自动同步到 `BBoxManager`。
  - **选中/移动**：点击已有框可选中（红色高亮），拖拽框体可移动位置。
  - **调整大小**：选中框右下角会出现红色小方块，拖拽可调整宽高。
  - **右键菜单**：在框上右键可选择"编辑标签"或"删除"。
  - **键盘删除**：按 `Delete` 键删除当前选中框。
  - **扩展虚线框**：如果右侧勾选了"预览扩展框"，每个 BBoxItem 的 `paint` 会额外绘制橙色虚线扩展框。
  - **结果标签**：每个框上方会绘制 `label | result` 文本，未标注时显示 `#索引号`。

### 4.2 control_panel.py（控制面板）
- **布局**：位于主窗口右侧，最大宽度 360px。
- **分区说明**：
  1. **模型选择**：下拉框（支持手动输入）+ "刷新模型"按钮。
  2. **系统提示词**：可编辑的多行文本框。
  3. **用户提示词**：可编辑的多行文本框。
  4. **Few-Shot / 系统设置**：快捷打开对应弹窗。
  5. **模型参数**：
     - Temperature（0 ~ 2，步进 0.1）
     - Top P（0 ~ 1，步进 0.05）
     - Max Tokens（1 ~ 8192）
     - Frequency Penalty（-2 ~ 2）
     - Presence Penalty（-2 ~ 2）
  6. **扩展倍数**：
     - 每个方向（中心、上、下、左、右）都有独立复选框和倍数输入框。
     - 勾选后才生效，倍数默认 1.0。
     - 勾选"预览扩展框"可在画布实时查看虚线扩展范围。
  7. **操作按钮**："开始送检"、"保存结果"。

### 4.3 file_list_widget.py（文件列表）
- **位置**：主窗口左侧。
- **功能**：显示当前导入的图片文件名列表，点击即可快速跳转到对应图片。与文件夹目录导入配合使用。

### 4.4 弹窗组件
- **settings_dialog.py**：设置 API 模式（cloud/local）、Base URL、API Key。
- **few_shot_dialog.py**：增删改 few-shot 示例对（User / Assistant）。
- **progress_dialog.py**：显示送检进度条（当前 / 总数）和取消按钮。

## 五、功能使用详解

### 5.1 打开图片
点击菜单栏 **文件 → 打开图片**，选择单张图片。系统将加载该图片到画布，并在状态栏显示路径和尺寸。

### 5.2 打开文件夹目录
点击菜单栏 **文件 → 打开文件夹**，选择一个包含图片的目录。系统会自动遍历目录下所有支持的图片格式（`.png`/`.jpg`/`.jpeg`/`.bmp`/`.gif`/`.tiff`/`.webp`），按文件名排序后加载到左侧文件列表。点击列表中的文件名即可切换图片。

### 5.3 系统提示词
在右侧控制面板的"系统提示词"文本框中输入内容。该提示词将作为 LLM 的 `system` 角色消息传入。修改后会随系统设置自动保存到 `settings.json`。

### 5.4 用户提示词
在右侧控制面板的"用户提示词"文本框中输入内容。该提示词将随裁剪后的图片一起作为 `user` 角色消息传入。

### 5.5 文件选择列表
打开文件夹后，左侧会列出所有图片文件名。点击任意文件名，主画布会立即切换到该图片，同时自动加载/保存对应的检测框数据（`_llm.json`）。

### 5.6 模型参数调整
右侧面板"模型参数"区域提供 5 个参数的精细调节：
- **Temperature**：控制输出随机性，值越高越随机。
- **Top P**：核采样阈值。
- **Max Tokens**：限制最大输出 token 数。
- **Frequency Penalty**：降低重复 token 的概率。
- **Presence Penalty**：鼓励模型讨论新话题。

### 5.7 图片预览窗口
打开图片后，中间大区域即为预览窗口。支持：
- 滚轮缩放
- 中键/拖拽平移
- 实时显示检测框、扩展虚线框、结果标签

### 5.8 图片编辑功能
菜单栏 **图像** 提供：
- **顺时针/逆时针旋转 90°**：对原图进行旋转并重新加载。
- **Resize...**：通过弹窗输入新的宽高像素值，对原图进行缩放。
- **重置显示**：将视图重置为自适应全图显示。

### 5.9 绘制矩形框
在画布空白处按住**鼠标左键拖拽**，即可绘制新的检测框。框的坐标基于原图像素，不受当前缩放比例影响。绘制后自动同步到 `BBoxManager`，并在框上方显示索引号。

### 5.10 模型选择
在右侧面板顶部的下拉框中选择模型。点击"刷新模型"按钮，系统会尝试从硅基流动 API 拉取最新模型列表。如果网络不可用，会自动回退到内置列表。你也可以直接在编辑框中手动输入模型 ID（如本地部署的模型名）。

### 5.11 系统设置
点击菜单栏 **文件** 旁的设置入口，或右侧面板的"系统设置"按钮，弹出设置窗口：
- **API 模式**：`cloud`（硅基流动）或 `local`（本地 OpenAI 兼容接口）。
- **云端设置**：填写硅基流动的 Base URL 和 API Key。
- **本地设置**：填写本地服务地址（如 `http://localhost:11434/v1`）和 API Key（通常为空）。
设置会持久化保存到 `settings.json`。

### 5.12 送检进度条
点击"开始送检"后，系统会弹出进度条窗口，显示当前已处理的目标数和总数。每个检测框对应一次 LLM 请求，按顺序依次执行。点击"取消"可随时中断。

### 5.13 复判结果显示
大模型返回的文本会写入对应检测框的 `result` 字段。画布会自动重绘，在框上方显示 `label | result` 标签。点击"保存结果"可将带标签的图片导出为 `{原图名}_result.{扩展名}`。

### 5.14 few-shot 添加窗口
点击右侧面板的"Few-Shot"按钮，弹出管理窗口。你可以添加多组 `User` 和 `Assistant` 的示例对话。送检时，这些示例会按顺序插入到最终请求的消息列表中，帮助大模型更好地理解任务格式。

### 5.15 送检按钮
点击"开始送检"后，系统执行以下动作：
1. 校验图片、检测框、模型是否已配置。
2. 按当前扩展倍数计算每个框的 ROI。
3. 依次裁剪 ROI 并调用 LLM API。
4. 将返回结果写回检测框并自动保存 `_llm.json`。

### 5.16 扩展倍数
在右侧面板"扩展倍数"区域：
- **中心**：以框中心为原点，等比例缩放整个框的宽高。
- **上 / 下 / 左 / 右**：分别向对应方向外扩。
- 倍数含义：`1.0` 表示不扩展，`2.0` 表示该方向长度变为原来的 2 倍。
- 支持多方向组合：例如同时勾选"上"2.0 倍和"左"1.5 倍，框会向上和向左分别扩展。
- 勾选"预览扩展框"后，画布会实时渲染橙色虚线框，便于确认范围。

## 六、数据持久化与自动保存

### 6.1 settings.json
位于项目根目录，保存：
- API 模式、URL、密钥
- 默认提示词、模型参数、Few-Shot 列表
- 上次选中的模型名

### 6.2 检测框数据
每张图片对应的检测框自动保存为同目录下的 `{原图名}_llm.json`，包含：
- `image_path`：原图路径
- `bboxes`：所有框的坐标、标签、复判结果

### 6.3 结果图片
点击"保存结果"后生成 `{原图名}_result.{扩展名}`，为带有框和标签标注的静态图片。

## 七、扩展性与维护性设计

- **松耦合**：`core` 层不依赖 `ui` 层，便于单元测试和后期替换 GUI 框架（如 PySide6、Web UI）。
- **统一 LLM 接口**：`llm_client.py` 屏蔽了不同厂商的协议差异，新增模型厂商只需修改一处。
- **信号驱动**：送检引擎通过 Qt 信号与 UI 通信，避免直接调用 UI 方法，降低线程安全风险。
- **配置外置**：所有默认值集中在 `config.py`，方便统一调整。
- **模型列表容错**：网络异常时无缝回退内置列表，保证系统可用性。
